# 技术方案 · 4 层架构与共享骨架

## 1. 总体架构

```
TeamWiki (纯静态)
├── 13 个模块页（每个 index.html ≤ 100 行）
├── 共享样式  styles/{base.css, wiki.css}
├── 共享数据  scripts/data.js     → window.WIKI
├── 共享骨架  scripts/layout.js   → 自动注入 nav + sidebar + footer
└── README.md
```

## 2. 共享骨架的工作机制

### `scripts/layout.js` 做的事

1. 读取 `window.WIKI`（从 `data.js`）。
2. 读取 `window.location.pathname` 判断当前页，匹配 sidebar 高亮。
3. 在 `<body>` 头部注入：液态光斑背景 + 顶部胶囊 nav。
4. 找到 `<main class="main">`，在其前面插入 sidebar，在其末尾插入 footer。
5. 注册全局 `⌘K` 快捷键 → 跳转 `/search/`。
6. 暴露 `window.toast(msg)` 给各模块使用。

### 新增模块只需 3 步

1. 建 `<新模块>/index.html`，引入 `base.css` + `wiki.css` + `data.js` + `layout.js`。
2. 在 `<body>` 内写 `<div class="app"><main class="main"> ... </main></div>`。
3. 在 `data.js` 的 `modules` 字典里加一条，指定路径 / 颜色 / icon。

无需手写 nav / sidebar / footer，layout.js 会自动注入。

## 3. 数据模型（当前）

```js
window.WIKI = {
  brand: { name, sub, tagline },
  layers: [{ id, name, en, color, desc, mods: [...modKey] }],
  modules: {
    [modKey]: { name, en, icon, color, path, count }
  },
  // 各模块的实际数据
  tools: [...], rules: [...], components: [...], prompts: [...],
  snippets: [...], builderOptions: {...}, cases: [...],
  knowledge: [...], usage: [...], agents: [...], people: [...]
};
```

## 4. Sprint 2 要演进的方向

- `window.WIKI` 拆分为 fetch 多个 `/data/*.json`，避免单文件过大。
- 接入真实 markdown：每个模块条目附带 `path` 字段，点击进入 `/viewer/?file=...` 渲染。
- 增加 `frontmatter` 支持：所有 `.md` 头部 yaml，含 `owner / tags / related / updated`。

## 5. 视图层与数据层的边界

- **视图层** = `teamwiki/`（HTML/CSS/JS） — 只负责渲染
- **数据层** = `knowledge/` 下的 markdown — 唯一真源
- **桥接** = `data.js`（当前手写）/ `data/*.json`（Sprint 2 由扫描脚本生成）

切工具不会断、个人迁移不会断的根因，是数据层独立于视图层。
