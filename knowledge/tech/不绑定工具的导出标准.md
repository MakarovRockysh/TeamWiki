# 技术方案 · 不绑定单一工具的导出标准

## 原则

同一份知识 / 规则，要能输出多种工具的原生格式，**切工具不会断**。

## 当前覆盖的工具

| 工具 | 期望文件 | 备注 |
|------|---------|------|
| Cursor | `.cursor/rules/*.mdc` | frontmatter 含 `description` / `globs` / `alwaysApply` |
| Codex (OpenAI) | `AGENTS.md` | 单一文件，章节式组织 |
| Claude Code | `CLAUDE.md` | 单一文件，可分小节 |
| 通用 LLM | `prompt.txt` 或粘贴文本 | 与上述去除工具特定语法 |

## 一份规则的源格式（建议）

```markdown
---
id: rule-design-tokens-v1
title: 设计 tokens 规范
scope: design
globs: ["**/*.tsx", "**/*.css"]
alwaysApply: true
related: ["rule-ui-glass-v1"]
owner: mzli
updated: 2026-05-03
---

# 设计 tokens 规范

## 颜色
- ...

## 间距
- ...
```

## 各工具的转换映射

### → `.cursor/rules/00-design-tokens.mdc`

完整保留 frontmatter，仅可选地按需要补字段，正文一字不动复制。

### → `AGENTS.md`

```markdown
## 设计 tokens 规范  (rule-design-tokens-v1)

> 适用：`**/*.tsx`, `**/*.css` · 始终生效

[正文]
```

每条规则一个二级标题，frontmatter 关键字段写入引用区块。

### → `CLAUDE.md`

与 `AGENTS.md` 类似，但可以按场景（设计 / PRD / 编码）分章节。

### → 通用 `prompt.txt`

```
[规则名]：设计 tokens 规范
[适用范围]：所有 .tsx 与 .css
[内容]：
- ...
```

去掉 markdown 语法，便于直接粘贴到任何 LLM 对话框。

## 转换逻辑放在哪里

- **当前 (Sprint 1)**：仅做格式预览，不真实生成文件。
- **Sprint 3**：Builder 选中规则后，前端实时生成 4 种格式的字符串，可下载 zip。
- **Sprint 5**：知识库 Agent 在每次更新源 markdown 后，自动同步生成 4 种格式到 `dist/exports/`。
