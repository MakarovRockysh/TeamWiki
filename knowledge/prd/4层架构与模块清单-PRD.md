# PRD · TeamWiki 4 层架构与模块清单

## 一、4 层架构

| 层 | 英文 | 职责 | 用户行为 |
|---|---|---|---|
| 资产层 | ASSETS    | 拿了就能用的现成东西 | 复制 / 下载 / 应用 |
| 沉淀层 | SEDIMENT  | 项目过程中产生的真实记录 | 翻阅 / 引用 / 学习 |
| 实践层 | PRACTICE  | 接进工作流的入口 | 触发 / 配置 / 监控 |
| 元层   | META      | 让 wiki 活下去的机制 | 搜索 / 探索 / 治理 |

每个模块只属于其中一层，避免"一堆并列的文件夹"。

## 三、设计原则

1. **不绑定单一工具**：所有资产可输出 `.cursor/rules/*.mdc` / `AGENTS.md` / `CLAUDE.md` / 通用 prompt。
2. **markdown 是真源**：原始文件留在 git，TeamWiki Web 只是视图层。
3. **每条都能用**：纯展示无操作的内容会被忽略，每个条目都内置"复制 / 引用 / 一键应用"出口。
4. **写 wiki 是工作的副产物**：靠 Agent 自动归档，不靠"组织号召大家写"。

## 四、模块演进策略

- 模块**只增不减**，但可以"打入归档"（不删，仅在 sidebar 隐藏）。
- 新增模块需先评估能否归入现有 4 层；不能则评估是否新增第 5 层。
- 单个模块内容超过 50 条时，引入二级筛选 / 分组。
