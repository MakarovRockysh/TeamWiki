# prd · 产品需求文档

存放 TeamWiki 自身的 PRD：模块 / 字段 / 交互 / 验收要点。

| 文件 | 用途 |
|------|------|
| `项目基础包构建器-PRD.md` | Builder 模块的需求规格 |
| `4层架构与模块清单-PRD.md` | 全站信息架构与 13 模块定义 |

## 写作规范

详见 `rules/prd-writing.md`（待补）。当前约定：

- 章节模板：背景 → 核心能力 → 输出物 → UI / 交互 → 字段说明 → 非目标 → 变更记录 → 关键词
- 文件名：`<模块名>-PRD.md`
- 必填关键词行（结尾），便于 grep
