# 2026-05-03 · 与 Sam 哥讨论"团队第二大脑"

## 结论 / 范围

1. **TeamWiki 的定位**：基础设施 ≠ 流水线。是把团队的脑子沉淀下来，价值随时间复利增长，不绑定单一工具。
2. **与 Sam 哥所做平台的对比**：Sam 哥在做"流水线"（产品级，AI 串起 PRD → 原型 → 开发 → 测试 → UAT），TeamWiki 做"基础设施"（团队第二大脑，资产沉淀）。两者互补，但路径不同。
3. **同一份知识 / 规则可输出多种格式**：`.cursor/rules/*.mdc`、`AGENTS.md`、`CLAUDE.md`、`prompt.txt` —— 切工具不会断。

## 五阶段闭环（Sam 哥讨论中明确的）

1. **项目初期** — 会议纪要沉淀（含人员关联）
2. **设计阶段** — 原型 / 流程设计（figma / stitch / cursor），同时维护设计规范 md
3. **编码阶段** — 编码规则 + 上下文存储脚本，约束 AI 不胡编
4. **测试阶段** — 测试 Agent 聚合 PRD/详设/接口，自动出用例
5. **运营阶段** — Cursor CLI 进驻服务器，监控运行，异常日志推送

## 未来两大核心方向（Sam 哥讨论中提出）

1. **强关系知识库**：除了团队复用，可成为可视化数据资产；长期甚至可作为企业评级、融资、贷款的参考依据。
2. **公司人群转型**：专业型岗位需求收敛，横轴复合型人才 × AI 能量倍增。

## 项目新建基础包（Sam 哥关心的具体落地点）

- 通过 TeamWiki 勾选技术底座 / 语言 / 组件 / 设计规范 / 规则限制
- 一键生成"项目新建基础包"链接
- Cursor 拉下来 → 自动形成项目结构

## 涉及文件 / 模块

- `teamwiki/index.html` — 4 层架构入口
- `teamwiki/builder/` — 项目基础包构建器（核心交互页）
- `teamwiki/knowledge/` — 本目录及兄弟目录
- `teamwiki/rules/` — 规则集（设计 / PRD / 输出 / 编码 + 写作元规范）
