# conversation-archive

本目录是 **TeamWiki** 项目下**唯一**的对话归档位置。用于存放可回顾、可检索的摘要，供后续会话上下文不足时查阅，并支持需求分析时对齐历史结论。

**助手约定**：除非另有说明，本会话助手生成的对话归档**一律**写入本目录（`knowledge/conversation/`），新建 `.md` 文件，勿无故覆盖已有归档。

## 触发归档（由你主动说）

- 当判断对话**差不多要占满上下文**时，可说：「**对话归档**」、「**归档当前对话**」、「**记一下**」等。
- 助手收到后应**自动**在本目录新建一个 `.md` 文件。
- **无法**由程序自动检测「上下文 90%」；以你的口头触发为准。

## 每个归档文件的建议结构

1. **标题**：一句话说明主题与日期（与文件名呼应）。
2. **结论 / 范围**：条目列出已定决策、已实现范围、待办。
3. **涉及文件或模块**：便于对照代码 / 知识库其他位置。
4. **检索关键词（必填）**：中英文、模块名、文件名等，便于 `grep` 与人工扫一眼。

## 文件命名

- 推荐：`YYYY-MM-DD-中文短主题.md`
- 或：`YYYY-MM-DD-short-topic-in-english.md`（英文 slug 避免路径兼容问题）

## 助手何时来读

- 你问"之前怎么定的"、"记不记得某功能"而当前对话里没有足够上下文时。
- 做**需求分析**或延续旧话题前，应优先在 **`knowledge/conversation/**/*.md`** 中按关键词搜索，再结合 `knowledge/prd/`、`knowledge/plan/`、`knowledge/tech/` 等交叉印证。

## 语言

- 正文使用**简体中文**，条目清晰，避免无意义流水账。
